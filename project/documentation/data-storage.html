Data Storage in MongoDB

Since the flow and functionalities in this web application require data to be displayed and verified, there was a need to have a database. MongoDB is being used as a NoSQL database for this application for backend storage. The tables are represented as collections and the rows(tupples) are represented as documents(JSON documents).
The following collections have been used for the project:

Collection Use in Project
FollowingSchema This collection is used to store the information about a particular user that the current user is following. The name of the user and the rating for that user is handled using this collection.
ProjectSchema This is the main collection in the project. It stores all the information about a user, i.e, login details, users that the current user is following with their rating.
MessagesSchema This collection is used to store all the messages. A message document would contain the sender, receiver, the time at which the message was sent and the message text itself. The messages for the currently logged in user(receiver) are searched amnd displayed in his "Messages" inbox.
