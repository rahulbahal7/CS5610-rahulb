Github API Introduction

About Experiment

This experiment will describe the behaviour of the Github API and resources that make up the official GitHub API v3.
All API access is over HTTPS, and accessed from the api.github.com domain. All data is sent and received in JSON format.

On the client side, the redirection is handled so that the user is navigated back to the Home page after Logging out of the application.

Pagination

Requests that return multiple items will be paginated to 30 items by default. You can specify further pages with the ?page parameter. For some resources, you can also set a custom page size up to 100 with the ?per_page parameter. Note that for technical reasons not all endpoints respect the ?per_page parameter, see events for example.
api.github.com/user/repos?page=3&per_page=100


The API can is used for requesting public information from Github. Some of these requests require authentication and/or callbacks. Some of the prominent API functionalities are:
  • Events and Notifications - Activity
  • Gists
  • Commits - Git Data
  • Issues
  • Organizations
  • Pull Requests
  • Repositories
  • Github Search
  • User Information
